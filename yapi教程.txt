#### yapi的结构

首先，yapi需要注册才可以使用。

yapi中有分组、项目、分类、接口这些层级。分组是项目的集合，项目中可以添加很多接口，多个接口可以通过分类来放在一起。

最高的是分组，只有超级管理员可以管理分组，分组组长可以也只能管理自己对应的分组。

分组下有项目和成员列表，点击某个分组，可以看到该分组下的项目和成员信息。

进入一个项目后，可以添加分类来管理接口，一个项目中可以有多个分类。

接口就是我们需要进行操作的单元了。



#### yapi的权限管理

对于一个团队，我们需要一个人来当超级管理员，只有他可以创建分组，将权限分配给分组组长，实际上管理各大分组与项目的便是分组组长。分组包括了项目和项目成员，每个项目中可以实现各自的接口。（超级管理员就相当于是老板，分组组长可以对应项目经理，产品经理，财务总监等等）

除了`个人空间`外,其他的`分组`可以邀请团队成员加入。只有`超级管理员`才有权限创建分组,其他用户无法创建分组。分组添加成功后,点击`添加成员`,邀请分组团队成员加入。添加成功后，被邀请的团队成员重新打开 yapi 平台，就可看到该分组。

#### 接口操作

- `接口集合` 将接口进行分类，使接口结构更清晰，一个接口只能属于一个集合，且不允许与其他接口重名。
- `测试集合` 为了方便我们测试接口，`测试集合` 将若干接口组合在一起，在这里一个接口可以属于不同集合。

##### 基本设置

- 接口路径：可以更改 HTTP 请求方式，并且支持 restful 动态路由，例如 /api/{id}/{name}, id和name是动态参数
- 选择分类：可以更改接口所在分类
- 状态：用于标识接口是否开发完成。
- Tag：用于标识接口tag信息（v1.3.23+）,在接口list页可以根据tag过滤接口

##### 请求参数设置

- Query参数： 接口 url 的查询字符串，点击『添加Query参数』按钮来添加参数，可以通过拖动来交换参数位置
- 请求Body：http 请求 body 部分，如果http请求方式是 post, put 等请求方式时会有 req_body 部分。req_body_type 形式有4种，分别是 form, json, file 和 raw 。
- Headers: http 请求头字段，在 req_body 形式是 form 格式下会在 header 中自动生成 'Content-Type application/x-www-form-urlencoded'，其他3种格式也会自动生成不同 header

##### 返回数据设置

- 返回数据分为 `json` & `raw` 两种形式。基于 mockjs （具体使用方法详见[Mock 介绍](https://hellosean1025.github.io/yapi/documents/mock.html)）和 json5，使用注释方式写参数说明。 为了方便数据编写可以按F9来使用全局编辑
- 选择json-schema 则进入了 json 结构可视化编辑器形式, 数据以 json schema 格式解析 [快速入门 Json Schema ](https://www.jianshu.com/p/8278eb2458c4?winzoom=1)。

##### 备注 & 其他

- 接口描述: 用简短的文字描述接口的作用。
- 邮件通知：开启后将此次接口的改动以邮件的形式发送至项目组所有成员和关注该项目的成员（邮件默认情况下自动开启）
- 开放接口：默认为关闭状态，用户可以在 数据导出 时选择只导出公开接口







































































## Q1 怎样联系组长？

组长分为 `分组组长` 和 `项目组长`:

- 分组组长：选择首页左侧的分组，点击右侧面板的 `成员列表`，成员右侧显示着 `组长/开发者` 的权限信息。 ![img](https://yapi.ymfe.org/documents/images/usage/manage_find_manager.png)
- 项目组长: 点击项目页的 `设置` - `成员列表`，成员右侧显示着 `组长/开发者` 的权限信息。 ![img](https://yapi.ymfe.org/documents/images/usage/manage_find_project_owner.png)
